## Workflow

1. Requester fills sections above "Engineering Review"
2. Submit by creating this file as `context/intake/[date]_[short_name].md`
3. Engineering reviews and either:
   - **Accept:** Creates PRP in `context/PRPs/`
   - **Needs Clarification:** Adds questions, returns to requester
   - **Reject:** Documents reason
4. Once PRP is approved and implemented, this intake is archived
